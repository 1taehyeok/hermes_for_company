# 파일 취급 정책

## 목적
원본 파일 훼손을 방지하고, 작업 이력을 안전하게 관리하기 위한 기본 규칙이다.

## 절대 원칙
- 원본 파일을 삭제하지 않는다.
- 원본 파일을 직접 수정하지 않는 것을 기본 원칙으로 한다.
- 수정이 필요하면 별도 작업폴더를 만들고 복사본으로 작업한다.

## 작업 방식
1. 원본 파일 위치를 확인한다.
2. 작업 목적별 폴더를 생성한다.
3. 원본 파일을 작업폴더에 복사한다.
4. 복사본에서만 수정 작업을 수행한다.
5. 결과물과 원본의 차이를 명확히 기록한다.

## 권장 폴더 예시
```text
work/
└─ 2026-04-20_시험의뢰접수_수정안/
   ├─ original_copy/
   ├─ working/
   └─ notes.md
```

## 기록 항목
- 작업 일시
- 작업자
- 원본 위치
- 복사본 위치
- 수정 목적
- 주요 변경 내용
- 검토 필요 여부

## 금지사항
- 원본 파일 덮어쓰기
- 임의 삭제
- 변경 이력 없는 수정
- 민감정보가 포함된 파일의 무분별한 공유

## 확인 체크리스트
- [ ] 원본 파일 삭제가 발생하지 않았는가
- [ ] 작업폴더가 분리되어 있는가
- [ ] 복사본 기준으로 수정했는가
- [ ] 변경 이유가 기록되어 있는가
