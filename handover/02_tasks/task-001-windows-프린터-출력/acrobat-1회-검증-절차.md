# Acrobat 1회 검증 절차

## 목적
중복/누적 출력 의심 상황에서 실제 출력 테스트를 **1회만** 수행하고, Windows 로그와 사용자 확인으로 결과를 닫는다.

## 기준점 확인
1. 대상 프린터 큐가 비어있는지 먼저 확인한다.
2. `Microsoft-Windows-PrintService/Operational` 로그가 켜져 있는지 확인한다.
3. 가능하면 테스트 직전 로그 기준점이 비어있거나 새 이벤트가 없는 상태에서 시작한다.

## 권장 테스트 조건
- PDF는 우선 **Acrobat /t**로 1회만 보낸다.
- 같은 파일로 연속 재시도하지 않는다.
- 테스트 직후 바로 사용자의 실물 출력 결과를 확인한다.
- 결과 확인 전 추가 출력 금지.

### 해석
- 이번 테스트에서는 **1회 요청 → 로그 1건 흐름 → 실물 1회 정상 출력**이 확인되었다.
- 따라서 Acrobat 경로가 항상 중복 출력되는 것은 아니다.
- 이후 비교 테스트에서 **한글 경로/파일명 킥보드 PDF는 실물 출력 실패**, 반면 **영문 작업본은 실물 출력 성공**이 확인되었다.
- 따라서 현재 운영에서는 **한글 경로/파일명 PDF를 직접 출력하지 않고, 영문 작업본을 만들어 출력하는 것**을 기본값으로 본다.

## 운영 원칙
1. 실물 테스트는 한 번만 보낸다.
2. 각 테스트 뒤에는 반드시 사용자에게 실제 출력 결과를 확인받는다.
3. 이상 증상이 있으면 재출력보다 먼저 로그/큐 상태를 다시 본다.
4. 원본 수정이 필요 없으면 직접 출력 경로를 우선한다.
5. fallback 출력은 사전 검증 없이 자동 실행하지 않는다.

## 메모
- `analysis-note-final.pdf` 영문 파일명 작업본 + Acrobat `/t` + X7600 조합은 이번 검증에서 정상 동작했다.
- Operational 로그는 추후 동일 이슈 재현 여부를 판단하는 1차 기준으로 쓴다.
